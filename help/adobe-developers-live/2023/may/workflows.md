# Découvrez les workflows et des conseils pour des déploiements plus rapides avec les environnements de développement rapide (RDE).

Découvrez les workflows et des conseils pour des déploiements plus rapides avec les environnements de développement rapide (RDE).

Les environnements de développement rapide (RDE) pour AEM as a Cloud Service améliorent l’expérience des développeurs en leur permettant de déployer et d’examiner rapidement les modifications, ce qui réduit le temps nécessaire pour tester les fonctionnalités. Les RDE prennent en charge les scénarios de développement itératifs, par exemple pour mettre à jour le code Java, les configurations, les scripts, le contenu et les configurations de Dispatcher. Les modifications sont directement appliquées à ces instances en cours d’exécution. Au cours de cette session, nous allons examiner les RDE en détail et montrer comment les développeurs peuvent en tirer parti pour déployer directement des modifications expérimentales, tester leur contenu et leur code pendant le développement et obtenir des commentaires rapides sur leurs modifications à l’aide d’un véritable environnement cloud. → Poursuivez la conversation à l’adresse : https://adobe.ly/3HACjKm
