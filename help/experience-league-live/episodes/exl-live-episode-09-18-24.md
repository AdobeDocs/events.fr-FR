# Êtes-vous prêt à mettre à niveau votre application mobile d’analyse ?

Découvrez comment utiliser l’extension Edge Bridge pour combler l’écart en matière de migration. Vous trouverez des instructions détaillées sur la configuration de vos propriétés mobiles et la mise à jour de votre implémentation côté client pour utiliser Edge Bridge.

>[!VIDEO](https://video.tv.adobe.com/v/3434575/?quality=12&learn=on)

**Pour poursuivre la discussion, consultez la [discussion sur la communauté Experience League](https://experienceleaguecommunities.adobe.com/t5/adobe-experience-platform/experience-league-live-post-session-discussion-are-you-ready-to/m-p/704990?profile.language=fr#M550){target="_blank"}.**


**Afficher les détails** :
Vous utilisez Adobe Analytics dans votre application mobile pour obtenir des informations sur votre entreprise numérique et comprendre comment vos clients utilisent votre application.

Vous avez également entendu dire que la collecte de données Edge est l’avenir de l’écosystème Adobe, et qu’elle vous permet d’exploiter d’autres solutions telles que Real-Time CDP, Journey Optimizer et Customer Journey Analytics.

Est-il possible de mettre à niveau votre application pour conserver l’implémentation existante pour Analytics et migrer vers Adobe Experience Platform (AEP) à votre propre rythme ?

Au cours de cette session, vous apprendrez à utiliser l’extension Edge Bridge pour combler le fossé en matière de migration. Nous vous fournirons des instructions détaillées sur la configuration de vos propriétés mobiles et la mise à jour de votre implémentation côté client pour utiliser Edge Bridge.

Après la migration, vous pouvez toujours accéder à vos données dans Analytics, ce qui vous rapproche d’AEP. Vous n&#39;êtes toujours pas sûr ? Nous vous montrerons comment valider votre implémentation et votre débogage en temps réel.
